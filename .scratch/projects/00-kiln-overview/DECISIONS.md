# Kiln Overview - Decisions

## Decisions Made

### Document Structure
**Decision:** Organize by concept → commands → configuration → integration
**Rationale:** Developers need to understand what Kiln is before learning specific commands, then how to configure it, then how to integrate

### Level of Detail
**Decision:** Include all command flags and options
**Rationale:** Developers need complete reference to understand integration points

### Include Custom Mode
**Decision:** Document Custom Mode but note it's experimental
**Rationale:** Bullish-ssg currently targets Default Mode, but developers should be aware of Custom Mode for future features

### Architecture Diagram
**Decision:** Include text-based architecture diagram
**Rationale:** Visual helps developers understand the relationship between bullish-ssg and Kiln

## No Blocked Decisions
All decisions were straightforward based on research.
